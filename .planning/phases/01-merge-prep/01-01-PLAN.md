1. 创建 `.gitattributes` 文件，配置以下 BMC 定制文件使用 `merge=ours` 策略（per PREP-02）:

```
# BMC 定制文件合并保护
# 在 git merge 时自动保留 BMC 版本，忽略上游变更
src/branding.ts merge=ours
src/providers/cos.ts merge=ours
src/providers/market.ts merge=ours
src/providers/index.ts merge=ours
src/telemetry.ts merge=ours
src/find.ts merge=ours
package.json merge=ours
README.md merge=ours
```

保护文件说明:
- `src/branding.ts` — BMC 品牌常量（包名 blueai-skills、URL、遥测端点）
- `src/providers/cos.ts` — BMC 独有的腾讯云 COS 存储提供者
- `src/providers/market.ts` — BMC 独有的技能市场提供者
- `src/providers/index.ts` — 包含 COS 和 Market 提供者的额外导出
- `src/telemetry.ts` — BMC 遥测默认关闭
- `src/find.ts` — BMC Market API 搜索实现
- `package.json` — BMC 包名、版本号、bin 字段
- `README.md` — BMC 中文文档

2. 配置 git merge driver:
```bash
git config merge.ours.driver true
```

`true` 命令总是返回 0，效果是在冲突时保留当前分支版本。

3. 将 .gitattributes 提交到当前分支:
```bash
git add .gitattributes
git commit -m "chore: 添加 .gitattributes 合并保护配置"
```
